# Platform Info Inventories

Merged inventories for permissions, feature flags, settings, access contracts, navigation, plan limits, and flag catalog entries.

**Agent skill:** `/rebuild-platform-info-inventories` · **Not** mandatory preload — see `.claude/skills/rebuild-platform-info-inventories/references/invoke-protocol.md`

## Authority ladder

1. **Runtime truth** — enforcing code (API routes, services, nav guards, UI gates)
2. **Declarative** — `web-admin/src/features/*/access/*-access.ts`, `web-admin/config/navigation.ts`, `web-admin/lib/constants/feature-flags.ts` (`FLAG_CATALOG`), DB permission/flag seeds
3. **Merged inventory** — `platform-info-inventory.json` (committed; generated by ingest)
4. **Human views** — `GENERATED_*.md` (script-only; never hand-edit)
5. **Legacy docs** — `docs/platform/permissions/*`, `feature_flags/*`, etc. — link here; do not duplicate tables by hand

## Commands

From repo root:

```bash
# Full pipeline (extract → ingest → reconcile → views)
npm run rebuild:platform-info-inventories

# Validate + access-contract Jest
npm run check:platform-info-inventories

# Ingest only (uses existing extracted-*.json)
npm run docs:ingest-platform-info
```

Strict drift (fail on new items): default for `npm run check:platform-info-inventories`. Local warn-only: `PLATFORM_INVENTORIES_WARN_ONLY=1 npm run check:platform-info-inventories`

## Help UI (admin)

Read-only browser for merged inventory data:

- **Route:** `/dashboard/help/platform-inventories` (requires `admin:read`)
- **API:** `GET /api/dev/platform-inventories?tab=contracts|permissions|flags|navigation|summary`
- **Shield link:** Top bar permissions dialog → “View full inventory in Help” (passes `?route=` filter)
