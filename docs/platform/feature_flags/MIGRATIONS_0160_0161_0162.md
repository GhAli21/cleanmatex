# Migrations 0160, 0161, 0162 — Settings Layers & B2B Self-Serve

Reference for settings resolver layers 3 & 4 and B2B_CONTRACTS_ENABLED tenant setting.

---

## Migration 0160 — PLAN_CONSTRAINT and FEATURE_FLAG Layers

**File:** `supabase/migrations/0160_stng_resolver_plan_constraint_and_feature_flag.sql`

**Purpose:** Implement Layer 3 (PLAN_CONSTRAINT) and Layer 4 (FEATURE_FLAG) in `fn_stng_resolve_setting_value`.

### Layer 3: PLAN_CONSTRAINT

- Reads tenant's plan from `org_pln_subscriptions_mst` or `org_tenants_mst.s_current_plan`
- Queries `sys_plan_setting_constraints` for the setting
- **deny:** If constraint exists and `constraint_value = true`, returns catalog default
- **max_value / min_value:** Caps the current value to plan limits

### Layer 4: FEATURE_FLAG

- Reads `stng_depends_on_flags` from `sys_tenant_settings_cd`
- For each flag in the array, calls `hq_ff_get_effective_value(p_tenant_id, flag_key)`
- If any flag returns falsy (false, 0, null, empty string): forces result to catalog default, sets `v_source_layer := 'FEATURE_FLAG'`, `v_source_id := 'DISABLED'`

**Dependencies:** 0151, 0074 (sys_plan_setting_constraints), 0068 (stng_depends_on_flags)

## Migration 0161 — B2B_CONTRACTS_ENABLED Setting

**File:** `supabase/migrations/0161_add_b2b_contracts_enabled_setting.sql`

**Purpose:** Tenant self-serve disable for B2B feature. When plan enables `b2b_contracts`, tenant can override to `false` via this setting.

### Catalog Entry (sys_tenant_settings_cd)

| Column | Value |
|--------|-------|
| setting_code | B2B_CONTRACTS_ENABLED |
| stng_depends_on_flags | ["b2b_contracts"] |
| stng_is_overridable | true |
| stng_default_value_jsonb | true |
| stng_scope | TENANT |
| stng_data_type | BOOLEAN |

### Resolution Flow

1. Plan enables `b2b_contracts` → FEATURE_FLAG layer allows B2B
2. Tenant sets override to `false` in `org_tenant_settings_cf` → TENANT_OVERRIDE disables
3. Resolution order ensures tenant wins

**Dependencies:** 0149 (B2B category), 0160 (FEATURE_FLAG layer)
