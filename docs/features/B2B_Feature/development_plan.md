# B2B Feature Development Plan

## Phases

| Phase | Scope | Est. Effort |
| --- | --- | --- |
| 1 | DB migrations, types, constants | 2–3 days |
| 2 | B2B customer CRUD, contacts, contracts | 4–5 days |
| 3 | Order/invoice integration, credit limit | 2–3 days |
| 4 | Statements, consolidated billing, dunning | 4–5 days |
| 5 | Cost center / PO (optional) | 1–2 days |
| 6 | Receipt channel rules | 1 day |
| 7 | Documentation | 1–2 days |

**Total**: ~16–21 days.

## Milestones

1. **Phase 1 complete**: Migration file created (user applies), types and constants in place
2. **Phase 2 complete**: B2B customers, contacts, contracts UI and APIs working
3. **Phase 3 complete**: Orders and invoices integrate with B2B; credit limit enforced
4. **Phase 4 complete**: Statement generation, PDF, dunning config
5. **Phase 5–7 complete**: Cost center, receipt rules, documentation
