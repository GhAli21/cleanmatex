# Documentation Canonicalization Skill

This skill resolves duplicated, overlapping, or conflicting documentation sources.

Use it when:

- multiple folders describe the same feature or domain
- a PRD folder and an active feature folder overlap
- you need to choose one canonical source of truth

Its job is to choose the winner, classify the others, and define a safe migration path.

Example trigger phrases:

- "which folder should be canonical?"
- "these docs overlap"
- "pick the source of truth for this domain"
