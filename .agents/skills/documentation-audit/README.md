# Documentation Audit Skill

This skill audits documentation health and completeness.

Use it when:

- you need a missing-file inventory
- you want readiness tiers or cleanup priorities
- you are reviewing one area or the whole repo for documentation gaps

Its output should be an audit, not a broad rewrite.

Example trigger phrases:

- "audit our feature docs"
- "what documentation is missing?"
- "give me a cleanup plan for docs"
