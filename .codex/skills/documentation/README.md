# Documentation Skill

This is the default lightweight documentation skill for ordinary CleanMateX documentation work.

Use it when:

- creating or updating feature docs
- filling a small number of missing pack files
- improving an already-canonical documentation folder

Do not use it for heavy governance work. Route to the specialist skills when needed:

- `documentation-canonicalization` for overlap and source-of-truth decisions
- `documentation-audit` for inventories and cleanup reports
- `documentation-pack-repair` for full pack completion in a known canonical folder
- `documentation-archive-migration` for legacy redirect/archive work

Example trigger phrases:

- "update this feature documentation"
- "add the missing user guide"
- "improve the docs for this feature"
