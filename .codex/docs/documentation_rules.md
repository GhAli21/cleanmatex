# Documentation Rules

## Authority Note

Use `docs/README.md` as the documentation hub authority.

This file is a helper summary for documentation behavior, not a competing structure definition.

## Current Rules

- keep one obvious canonical folder for each feature family
- separate active docs from historical material where possible
- if full physical relocation is not yet safe, add clear cross-links and grouped indexes
- update module READMEs, feature READMEs, and planning references to match implemented reality
- avoid placeholder text in active entrypoint docs

## What To Document

- feature overview and scope
- implementation status
- active versus historical role
- related canonical docs
- permissions, settings, i18n, APIs, migrations, and limits where relevant
