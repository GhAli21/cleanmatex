# Documentation Pack Repair Skill

This skill completes or repairs the standard documentation pack inside a known canonical folder.

Use it when:

- the correct feature folder is already chosen
- many standard pack files are missing
- you want consistent pack completion with minimal churn

If canon is still unclear, use `documentation-canonicalization` first.

Example trigger phrases:

- "complete this documentation pack"
- "fill the missing pack files"
- "repair this feature folder docs"
